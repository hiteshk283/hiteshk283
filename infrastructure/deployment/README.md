# Deployment Infrastructure

This directory documents the CI/CD pipelines and the static hosting infrastructure for the Control Center.

## Static Frontend Hosting (AWS S3 & CloudFront)

Instead of running a web server to host the Flutter Web app, we use AWS's static website hosting capabilities:

1. **Amazon S3**:
   - Bucket: `control-center-frontend-hiteshk283`
   - Stores the compiled HTML, JS, and CSS files generated by `flutter build web`.
2. **Amazon CloudFront**:
   - Distribution ID: `E2ALGJR4OOW90A`
   - A global Content Delivery Network (CDN) that caches the S3 bucket's content at edge locations worldwide, ensuring ultra-fast load times for users.
   - Provides HTTPS out of the box.

## CI/CD Pipelines (GitHub Actions)

All deployments are fully automated using GitHub Actions (located in `.github/workflows/`). They are configured to run manually via `workflow_dispatch`.

1. **Frontend Pipeline**:
   - Checks out the code and sets up the Flutter environment.
   - Builds the Android Release APK and uploads it as a GitHub Artifact.
   - Builds the Flutter Web app.
   - Syncs the web build directory to the Amazon S3 bucket.
   - Invalidates the CloudFront cache so users immediately see the new version.

2. **Backend Pipeline**:
   - Checks out the code and logs into Amazon ECR.
   - Builds a new Docker image from the `backend/` directory.
   - Pushes the image to ECR with a unique commit SHA tag.
   - Downloads the active ECS Task Definition and updates the image URI.
   - Deploys the new task definition to the ECS Service, triggering a rolling update.
